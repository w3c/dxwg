Abstract

A vocabulary for listing common roles played by Resource Descriptor objects within the Profiles Ontology [[PROF]].

The namespace for PROF terms is http://www.w3.org/ns/dx/prof/role/.
This vocabulary, defined in OWL and encoded in RDF Turtle, is available at resource_roles.ttl.

Overview of DXWG documents on profiles

This document is part of a set of documents on profiles, edited by the W3C Dataset Exchange Working Group (DXWG). Some of the documents are general while some are technology-specific:

  • [[PROF-GUIDANCE]], the top-level general recommendations and guidance on profiling. It gives an overview of all DXWG documents on profiles and is the recommended starting point
  • [[PROF]], an RDF vocabulary that describes profiles and related resources
  • [[PROF-CONNEG]], the specific guidance on how to negotiate for Internet resource content using profiles
  • [[PROF-IETF]], an IETF Internet-Draft defining HTTP Headers for HTTP content negotiation by profile
  • [PROF-ROLES] (this document), a vocabulary of Profiles Vocabulary Resource Role instances

Introduction

This vocabulary contains a controlled list of prof:ResourceRole instances, which are also skos:Concepts, for use with the Profiles Vocabulary [[PROF]].

It is suggested, and expected, that this vocabulary be extended by users to suite their own needs.

Namespaces

The namespace for this vocabulary is http://www.w3.org/ns/dx/prof/role/ which is derived from the namespace of the Profiles Vocabulary [[PROF]], http://www.w3.org/ns/dx/prof/.

A set of namespaces and prefixes used for them in this document is shown in the table below.

Motivation

In discussions during the creation of the Profiles Vocabulary, it was apparent that there were a few Resource Roles that were common to many profiling scenarios. A vocabulary to manage these common Roles for maximal reuse thus seemed to be a useful Working Group output.

This list will grow over time.

Vocabulary

Role: Aggregated Constraints

SKOS Concept:role:aggregatedConstraints
Pref Label:Aggregated Constraints
Definition:Full Constraints not only for this Profile but also for all dependencies (Profiles or dct:Standards)
Usage Note:Use this Role when you want to provide a resource that validates data according to this profile and also its dependents. You must include the Full Constraints of all dependencies within this resource
Broader:role:fullConstraints

SKOS Concept:role:codeRepository
Pref Label:Code Repository
Definition:Repository for source code and profile artifacts

Role: Extension Constraints

SKOS Concept:role:extensionConstraints
Pref Label:Extension Constraints
Definition:Constraints implemented by this Profile building on dependencies but not including them
Usage Note:Use this Role when you want to provide a resource that validates data according to this profile but only in so far as it is a constraint on its dependents. Therefore, this resource must not also include any dependents' constraints
Broader:role:partConstraints
